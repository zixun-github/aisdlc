# QA 测试规划器

面向 QA 工程师的综合技能，用于创建测试计划、生成手工测试用例、构建回归测试套件、对照 Figma 验证设计，并有效记录缺陷。

> **激活方式：** 本技能仅在显式按名称调用时触发（例如 `/qa-test-planner`、`qa-test-planner` 或 `use the skill qa-test-planner`）。

---

## 快速开始

**创建测试计划：**
```
"为用户认证功能创建测试计划"
```

**生成测试用例：**
```
"为结账流程生成手工测试用例"
```

**构建回归套件：**
```
"为支付模块构建回归测试套件"
```

**对照 Figma 验证：**
```
"将登录页与 Figma 设计 [URL] 进行对比"
```

**创建缺陷报告：**
```
"为表单验证问题创建缺陷报告"
```

## 核心交付物

### 1. 测试计划
- 测试范围和目标
- 测试方法和策略
- 环境要求
- 准入/准出标准
- 风险评估
- 时间线和里程碑

### 2. 手工测试用例
- 分步说明
- 预期 vs 实际结果
- 前置条件和准备
- 测试数据要求
- 优先级和严重程度

### 3. 回归套件
- 冒烟测试（15-30 分钟）
- 完整回归（2-4 小时）
- 定向回归（30-60 分钟）
- 执行顺序和依赖关系

### 4. Figma 验证
- 逐组件对比
- 间距和排版检查
- 颜色和视觉一致性
- 交互状态验证

### 5. 缺陷报告
- 清晰的复现步骤
- 环境详情
- 证据（截图、日志）
- 严重程度和优先级

## 反模式

| 避免 | 原因 | 替代做法 |
|------|------|----------|
| 模糊的测试步骤 | 无法复现 | 具体操作 + 预期结果 |
| 缺少前置条件 | 测试意外失败 | 记录所有准备要求 |
| 无测试数据 | 测试人员受阻 | 提供示例数据或生成方式 |
| 泛化缺陷标题 | 难以追踪 | 具体化："[功能] 在 [操作] 时出现问题" |
| 跳过边界情况 | 遗漏关键缺陷 | 包含边界值、空值 |

### 严重程度定义

| 级别 | 标准 | 示例 |
|------|------|------|
| **严重 (P0)** | 系统崩溃、数据丢失、安全 | 支付失败、登录不可用 |
| **高 (P1)** | 主要功能损坏、无变通方案 | 搜索不可用 |
| **中 (P2)** | 功能部分可用、有变通方案 | 筛选器缺少一个选项 |

### 设计验证工作流

**前置条件：**
- Figma MCP 服务器已配置
- 可访问 Figma 设计文件
- 组件/页面的 Figma URL

**流程：**

1. **从 Figma 获取设计规格**
```
"从 Figma 文件 [URL] 获取按钮规格"

响应包含：
- 尺寸（宽、高）
- 颜色（背景、文字、边框）
- 排版（字体、大小、粗细）
- 间距（内边距、外边距）
- 圆角
- 状态（默认、悬停、激活、禁用）
```

2. **对比实现**
```
TC: 主按钮视觉验证
1. 在浏览器开发者工具中检查主按钮
2. 与 Figma 规格对比：
   - 尺寸：120x40px
   - 圆角：8px
   - 背景色：#0066FF
   - 字体：16px Medium #FFFFFF
3. 记录差异
```

3. **不匹配时创建缺陷**
```
BUG: 主按钮颜色与设计不符
严重程度：中
预期（Figma）：#0066FF
实际（实现）：#0052CC
截图：[已附]
Figma 链接：[具体组件]
```

### 验证内容

| 元素 | 检查项 | 工具 |
|------|--------|------|
| 颜色 | 十六进制值精确 | 浏览器取色器 |
| 间距 | 内边距/外边距 px | DevTools 计算样式 |
| 排版 | 字体、大小、粗细 | DevTools 字体面板 |
| 布局 | 宽、高、位置 | DevTools 盒模型 |
| 状态 | 悬停、激活、聚焦 | 手工交互 |
| 响应式 | 断点行为 | DevTools 设备模式 |

### 示例查询
```
"从 Figma 设计 [URL] 获取按钮规格"
"将导航菜单实现与 Figma 设计对比"
"从 Figma 提取仪表板布局的间距值"
"列出 Figma 设计系统中使用的所有颜色令牌"
```

### 套件结构

| 套件类型 | 时长 | 频率 | 覆盖 |
|----------|------|------|------|
| 冒烟 | 15-30 分钟 | 每日 | 仅关键路径 |
| 定向 | 30-60 分钟 | 每次变更 | 受影响区域 |
| 完整 | 2-4 小时 | 每周/发布 | 全面 |
| 健全性 | 10-15 分钟 | 热修复后 | 快速验证 |

### 构建回归套件

**步骤 1：识别关键路径**
- 用户不可或缺的功能？
- 产生收入的功能？
- 处理敏感数据的功能？
- 使用最频繁的功能？

**步骤 2：测试用例优先级**

| 优先级 | 描述 | 必须执行 |
|--------|------|----------|
| P0 | 业务关键、安全 | 始终 |
| P1 | 主要功能、常见流程 | 每周+ |
| P2 | 次要功能、边界情况 | 发布时 |

**步骤 3：执行顺序**
1. 先冒烟 - 若失败，停止并修复构建
2. 其次 P0 测试 - 通过后才能继续
3. 然后 P1、P2 - 记录所有失败
4. 探索性 - 发现意外问题

### 通过/失败标准

**通过：**
- 所有 P0 测试通过
- 90%+ P1 测试通过
- 无未解决的严重缺陷

**失败（阻止发布）：**
- 任一 P0 测试失败
- 发现严重缺陷
- 安全漏洞
- 数据丢失场景

**有条件：**
- P1 失败但有变通方案
- 已知问题已记录
- 修复计划已就位

<summary><strong>最佳实践</strong></summary>

### 测试用例编写

**应做：**
- 具体且无歧义
- 每步包含预期结果
- 每个用例只测一件事
- 使用一致的命名规范
- 保持用例可维护

**勿做：**
- 假设已知
- 用例过长
- 跳过前置条件
- 忽略边界情况
- 预期结果模糊

### 缺陷报告

**应做：**
- 提供清晰的复现步骤
- 附上截图/视频
- 指定准确的环境详情
- 描述对用户的影响
- UI 缺陷附 Figma 链接

**勿做：**
- 无复现步骤就报告
- 使用模糊描述
- 跳过环境详情
- 忘记分配优先级
- 重复已有缺陷

### 回归测试

**应做：**
- 尽可能自动化重复测试
- 定期维护回归套件
- 优先关键路径
- 频繁执行冒烟测试
- 每次发布后更新套件

**勿做：**
- 发布前跳过回归
- 让套件过时
- 每次都测全部
- 忽略失败的回归测试

</details>

## TC-LOGIN-001: 有效用户登录

**优先级：** P0（严重）
**类型：** 功能
**预估时间：** 2 分钟

### 目标
验证用户可使用有效凭据成功登录

### 前置条件
- 用户账号存在（test@example.com / Test123!）
- 用户未已登录
- 浏览器 Cookie 已清除

### 测试步骤
1. 访问 https://app.example.com/login
   **预期：** 登录页显示邮箱和密码输入框

2. 输入邮箱：test@example.com
   **预期：** 邮箱框接受输入

3. 输入密码：Test123!
   **预期：** 密码框显示掩码字符

4. 点击「登录」按钮
   **预期：**
   - 显示加载指示器
   - 用户重定向至 /dashboard
   - 显示欢迎消息：「欢迎回来，Test User」
   - 页头显示头像/个人资料图片

### 后置条件
- 用户会话已创建
- 认证令牌已存储
- 分析事件已记录

### 需考虑的边界情况
- TC-LOGIN-002：无效密码
- TC-LOGIN-003：不存在的邮箱
- TC-LOGIN-004：SQL 注入尝试
- TC-LOGIN-005：超长密码
```

## TC-UI-045: 移动端导航菜单

**优先级：** P1（高）
**类型：** UI/响应式
**设备：** 移动端（iPhone、Android）

### 目标
验证导航菜单在移动设备上正确工作

### 前置条件
- 从移动设备或响应式模式访问
- 视口宽度：375px（iPhone SE）至 428px（iPhone Pro Max）

### 测试步骤
1. 在移动设备上打开首页
   **预期：** 汉堡菜单图标可见（右上角）

2. 点击汉堡图标
   **预期：**
   - 菜单从右侧滑入
   - 内容上出现遮罩
   - 关闭（X）按钮可见

3. 点击菜单项
   **预期：** 导航至对应区块，菜单关闭

4. 与 Figma 移动端设计 [链接] 对比
   **预期：**
   - 菜单宽度：280px
   - 滑动动画：300ms ease-out
   - 遮罩透明度：0.5，颜色 #000000
   - 字体大小：16px，行高 24px

### 需测试的断点
- 375px（iPhone SE）
- 390px（iPhone 14）
- 428px（iPhone 14 Pro Max）
- 360px（Galaxy S21）
```
